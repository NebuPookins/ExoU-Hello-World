Hello World

This is a minimal application to show you how to create an EXO U application.

EXO U Applications are stored in your EXOapps Folder. Depending on what OS you're using, your EXOapps folder is probably in one of these locations:

Microsoft Windows
C:\Users\[YOUR USER]\Documents\My Apps\EXOapps\
Google Android
/mnt/sdcard/Android/data/exopc.engine/files/EXOapps
Apple Mac OS
/Users/USERNAME/EXOapps/
Linux
/home/USERNAME/EXOapps/

This application is in a subfolder called Nebu_HelloWorld. You can view the contents of that folder to see how this app was implemented.

To create a new application, create a subfolderin the EXOapps folder, then create an html file with the same name as the subfolder, but with the string app- prepended to it. For example, if your folder is called MyExoUApp, then your html file should be called app-MyExoUApp.htm. Note also that the extension must be .htm; .html will not work.

Your application must declare a globally visible exomain JavaScript function. The EXO U framework will invoke that function after your HTML page has loaded. The EXO U framework also exposes a globally visible exo object as its API. The documentation for the API is available at http://www.exou.com/api-object-reference/.

Your code should set the width and height of the exo.app object, because by default, the width and height are 0, meaning your application will not be visible to the user. For example, this app uses the following code: 

function exomain() { 
	exo.app.width = exo.workspace.width;
	exo.app.height = exo.workspace.height;
	exo.app.left = 0;
	exo.app.top = 0;
	exo.app.movable = 0;
	var scrollable_container = document.getElementById("scrollable_container");
	scrollable_container.style.height = exo.app.height + "px";
}

The last two lines are related to the fact that by default, the HTML renderer does not display a scrollbar if your application's contents exceed the height of the viewport.

To customize the icon for the app, create a PNG file named app.png. The icon for this particular app was designed by Svengraph and is used with the "You can share these icons with url back to Officinadigitale.org (credits)" license.

After making changes to your code (e.g. in a text editor), you can simply alt tab back into EXO, quit your app, and then run the app again to see the changes. I.e., you do not need to restart EXO, nor do you need to tell it to "refresh the apps" or anything like that; it will reload the code each time the application starts up.
